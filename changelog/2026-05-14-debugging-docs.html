Added Debugging section to README and agent pointer in AGENTS.md

docs 2026-05-14 · commit b2f8fdb

Why

The repo had end-to-end deploy + verify, but if something went wrong post-deploy there was no documented place for the user or a future agent to start. Without a single "where do I look first" reference, every troubleshooting session would have to rediscover the log locations, state file format, and command incantations.

What changed

  • README.md — new Debugging section. Six subsections:
    • Where to look first (logread -e ont-watchdog, cron-firing check, state file, active config).
    • Tasmota inspection commands (Status 0/1/5, BootCount tracking, web log endpoint).
    • Log line glossary — table mapping every ont-watchdog: message to its meaning.
    • State file format reference.
    • Symptom → check table for 8 common failure modes.
    • Forcing scenarios for testing (manual power cycle, state reset, dry-run, blackhole simulation).
    • Log retention caveat — logread is in-RAM.
  • AGENTS.md — added a Debugging primer pointing back to README with two go-to commands. Surfaces the note that climbing BootCount is a power/signal issue, not a logic bug.

Verification

Docs-only change; no deploy, no behavior modification. Reviewed rendered Markdown for table alignment.
