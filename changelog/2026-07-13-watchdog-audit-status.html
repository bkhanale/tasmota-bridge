Added durable watchdog event audit, action-state recovery, and status snapshot

fix 2026-07-13

Why

A recent WAN outage confirmed the watchdog could recover service, but the detailed evidence lived only in OpenWrt’s small in-RAM log buffer. It also exposed that losing the tmpfs runtime state could discard the action counter and allow a repeat cycle before the intended escalating backoff.

What changed

  • router/ont-watchdog.sh now keeps a bounded, event-only audit at /root/ont-watchdog-events.log, rotating at 32 KiB with three previous generations. It records outage starts, actions, recovery, and runtime-state restoration, not ordinary minute runs.
  • Completed action timing is backed up to /root/ont-watchdog.action-state. If /tmp/ont-watchdog.state is missing, the watchdog restores cooldown and rate-limit information before deciding whether it may cycle again.
  • status.sh and deployed /root/ont-watchdog-status.sh provide a read-only snapshot of relay state, cron, both state files, and recent durable events.
  • deploy.sh, verify.sh, README.md, and AGENTS.md deploy, verify, and document the new operational surfaces.

Verification

Ran shell syntax and whitespace checks, deployed with ./deploy.sh, ran the end-to-end verifier, and observed a subsequent scheduled cron invocation. The status helper was included in deployment verification.

Rollback

Revert this commit and run ./deploy.sh. Optionally remove /root/ont-watchdog-events.log* and /root/ont-watchdog.action-state from the router afterward.
